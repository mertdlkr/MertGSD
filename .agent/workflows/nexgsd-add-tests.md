# NexGsd Add Tests

Use this after implementation when test coverage is weak.

## Steps

1. Identify the target area from arguments or recent work.
2. Read relevant source files and existing test files.
3. Add the smallest useful set of tests that proves the intended behavior.
4. Prefer real behavior checks over shallow snapshots unless snapshots are already the project norm.
5. Run the relevant test command and report the result.

## Completion

Report added tests, what they prove, and any remaining gaps.
