# NexGsd New Milestone

Brownfield equivalent of `/nexgsd-new-project`.

## Steps

1. Read current `PROJECT.md`, `ROADMAP.md`, `REQUIREMENTS.md`, and `STATE.md`.
2. Ask what should ship next.
3. Decide milestone name/version with the user.
4. Update project context for the new milestone.
5. Add or refresh milestone-scoped requirements.
6. Extend `ROADMAP.md` with the new milestone phases.
7. Reset current position in `STATE.md` to the new milestone start.
8. Commit milestone kickoff docs if planning docs are tracked.

## Completion

Recommend:

- `/nexgsd-discuss [first new phase]`
- or `/nexgsd-plan [first new phase]`
